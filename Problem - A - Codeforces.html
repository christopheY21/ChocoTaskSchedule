A. Task Scheduling and Data Assignment
time limit per test
15 seconds
memory limit per test
1024 megabytes
input
standard input
output
standard output

With the rapid development of chip hardware computing capabilities, hardware architectures are evolving towards more complex multi-core and heterogeneous architectures. In this problem, your goal is to design an efficient task scheduling algorithm together with a memory allocation policy for the software-hardware co-optimization problem that exists widely in operating systems, distributed systems, and compilers.

Description

There are ℓℓ tasks that need to be scheduled on nn machines with varying processing powers. There are also mm disks with varying transmission speeds and storage capacities for storing the data generated by each task after its execution.

Task schedule. A task schedule is an assignment of tasks to machines and disks. More specifically, for each task ii, a task schedule specifies three parameters (xi,yi,zi)(xi,yi,zi), meaning that ii starts at time xixi on machine yiyi and stores its data to disk zizi. The earliest starting time of any task is 0.

Task Running Process Here are more details about the running process of a task ii when the task schedule is fixed.

  • The processing power of machine jj is power(j)power(j). Each task ii has a size of size(i)size(i), which means it needs ⌈size(i)/power(yi)⌉⌈size(i)/power(yi)⌉ units of time to execute when it is run on machine yiyi. Here, ⌈.⌉⌈.⌉ means rounding up.
  • Each disk kk has a read/write transmission rate of speed(k)speed(k). When a task ii finishes its execution, it will generate a piece of data of size data(i)data(i) that needs to be stored on some disk. This storing step will take ⌈data(i)/speed(zi)⌉⌈data(i)/speed(zi)⌉ units of time if this data is stored on disk zizi. Here, the value of time also needs to be rounded up.
  • Before a task ii starts its execution, it needs to read the output data of some other tasks. We let PREDdata(i)PREDdata(i) denote the set of tasks whose output data are needed by task ii. This will take a total time of ∑j∈PREDdata(i)⌈data(j)/speed(zj)⌉∑j∈PREDdata(i)⌈data(j)/speed(zj)⌉ and this must be done before task ii starts its execution. Note that when calculating the total time, each term needs to be rounded up individually and then summed.

Summarizing the above process, when task ii starts, it goes through three phases: reading data from other tasks, executing the task, and storing data. For convenience, we denote the starting time of the three phases as ai,bi,ciai,bi,ci and the finishing time of the last phase as didi. These values are determined as follows.

  • ai=xiai=xi
  • bi=ai+∑j∈PREDdata(i)⌈data(j)/speed(zj)⌉bi=ai+∑j∈PREDdata(i)⌈data(j)/speed(zj)⌉.
  • ci=bi+⌈size(i)/power(yi)⌉ci=bi+⌈size(i)/power(yi)⌉.
  • di=ci+⌈data(i)/speed(zi)⌉di=ci+⌈data(i)/speed(zi)⌉.

In addition, a feasible task schedule should satisfy the following requirements:

  • No preemption: Each machine executes only one task or transmits one data at a time, and no interruption is allowed during its lifecycle. In other words, for each pair of tasks ii and jj such that yi=yjyi=yj, the two intervals (ai,di)(ai,di) and (aj,dj)(aj,dj) cannot have any overlaps.
  • Task dependencies: Each task ii has a list of task-dependent tasks, denoted by PREDtask(i)PREDtask(i). For each task j∈PREDtask(i)j∈PREDtask(i), task ii cannot start until jj finishes its execution. That is, it must satisfy ai≥cjai≥cj. Note that if ii is scheduled on the same machine with jj, then it still needs to wait for jj to complete storing its data.
  • Data dependencies: For each task j∈PREDdata(i)j∈PREDdata(i), task ii cannot start until jj finishes storing its data. That is, it must satisfy ai≥djai≥dj.
  • Affinity of machines: Each task ii has a list of affinitive machines. denoted by AiAi, that it can be assigned to. Each task must be scheduled on one of its affinitive machines. That is, it must satisfy yi∈Aiyi∈Ai.
  • Disk Capacity: Each disk kk has a storage capacity of capacity(k)capacity(k). The total size of all data stored on the same disk cannot exceed that disk's capacity. That is, for each disk kk it must satisfy ∑i:zi=kdata(i)≤capacity(k)∑i:zi=kdata(i)≤capacity(k).

Given a feasible schedule, the makespan of this schedule is the finishing time of the last task. In other words, makespan=max1≤i≤ℓdimakespan=max1≤i≤ℓdi

Objectives

Your objective is to find a feasible task schedule that minimizes the makespan.

Input

The first line contains an integer ℓℓ representing the number of tasks. In the following ℓℓ lines, each line represents a task: The first three integers represent task id ii, task size size(i)size(i), and the size of its output data data(i)data(i). All task ids are distinctive integers between 11 and ℓℓ. The fourth integer kk is the number of affinitive machines for this task, and the remaining kk integers are the IDs of these affinitive machines and they are distinctive as well.

Next, a line contains an integer nn representing the number of machines. In the following nn lines, each line has two integers representing machine id jj and its power power(j)power(j). All machine ids are distinctive integers between 11 and nn.

Next again, a line contains an integer mm representing the number of disks. In the following mm lines, each line has three integers representing disk id kk, its speed speed(k)speed(k), and capacity capacity(k)capacity(k). All disk ids are distinctive integers between 11 and mm.

The remaining part describes the two types of dependencies between tasks.

The first line of this part is an integer NN representing the number of data dependencies. In the following NN lines, each line contains two integers i,ji,j, which means task jj is data-dependent on task ii.

Next, there is a new line with an integer MM representing the number of task dependencies. In the following MM lines, each line contains two integers i,ji,j, which means task jj is task-dependent on task ii.

All numbers on the same line are separated by whitespaces.

Variable constraints

  • number of tasks: 6≤l≤100006≤l≤10000
  • number of machines: 1≤n≤501≤n≤50
  • number of disks: 2≤m≤302≤m≤30
  • size of task: 10≤size(i)≤60010≤size(i)≤600
  • size of outputted data: 0≤data(i)≤200≤data(i)≤20
  • computing power of machines: 1≤power(i)≤201≤power(i)≤20
  • capacity of disks: 1≤capacity≤10500001≤capacity≤1050000
  • rate of disks: 1≤speed≤201≤speed≤20
  • all the ids are consequent and starts from one.
  • the dependence graph which includes both two types of dependence edge is acyclic.
  • size of dependence: M+N≤500000M+N≤500000
Output

You should output ℓℓ lines describing the task schedule for all tasks. The iith line should contain four integers i,xi,yi,zii,xi,yi,zi, separated by whitespaces, meaning that task ii will start at time xixi on machine yiyi and store its data on disk zizi.

Scoring

For each test, the score is the lower bound divided by your makespan, where the lower bound is computed via

100⋅(∑i∈[l]size(i)/∑j∈[n]power(j)+∑i∈[l](data(i)⋅(outdegree(i)+1))/∑j∈[m]speed(j)).100⋅(∑i∈[l]size(i)/∑j∈[n]power(j)+∑i∈[l](data(i)⋅(outdegree(i)+1))/∑j∈[m]speed(j)).
where, outdegree(i)outdegree(i) means the number of edges that go out from task ii in DAG (In other words, it is the number of tasks that depends on task ii).

We guarantee that all the cases have feasible solutions since there is a disk of rate 11 with a capacity greater or equal to the sum of data(i)data(i) over all tasks ii. If you output an invalid solution, you get zero. Here, makespan stands for the latest end-time of all tasks when the earliest start time of all tasks is zero. The final score is the summation of all tests' scores.
